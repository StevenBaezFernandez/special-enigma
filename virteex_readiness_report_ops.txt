
## 3. Ops Dashboard & Gestión de Tenants (Soporte Interno)

La aplicación `virteex-ops` es crucial para que el equipo de soporte resuelva problemas de clientes sin acceder a la base de datos directamente, pero está incompleta.

### 3.1. Gestión de Tenants (Incompleto)
- **TenantsComponent:**
    - **Estado:** Muestra una tabla básica de tenants. El botón "New Tenant" tiene un `// TODO: Implement navigation`.
    - **Impacto:** Soporte no puede crear cuentas manualmente para clientes Enterprise o corregir datos erróneos de onboarding.
    - **Acción:** Implementar `CreateTenantComponent` (formulario reactivo completo) y conectarlo al endpoint `POST /admin/tenants` del `virteex-admin-service`.
- **TenantService (Mock):**
    - **Estado:** `TenantService` en Ops usa `Observable` de datos mockeados o una llamada HTTP simple sin manejo de errores robusto.
    - **Mejora:** Implementar `Search` y `Filter` (por país, estado, plan) en el backend (`AdminService`) para que el frontend pueda paginar eficientemente miles de tenants.

### 3.2. Autenticación de Operadores
- **AuthService (Ops):**
    - **Estado:** Tiene un mock hardcodeado: `const mockUser = { id: 1, email: credentials.email, role: 'admin', token: 'mock-jwt-token' };`.
    - **Riesgo:** CUALQUIERA puede loguearse al panel de operaciones con cualquier credencial en este momento.
    - **Acción URGENTE:**
        - Conectar `AuthService` de Ops al `IdentityService` real (usando un cliente `ClientCredentials` o flujo `Resource Owner Password` restringido a IPs internas/VPN).
        - Implementar Roles RBAC estrictos (`SUPER_ADMIN`, `SUPPORT_L1`, `SUPPORT_L2`) en el token JWT para limitar acciones destructivas (borrar tenant, ver facturas sensibles).
